# 순례길 내비게이션 독립 PWA

가톨릭길동무 본앱에 나중에 병합하기 위해 분리한 순례길 내비게이션 전용 PWA입니다.

## 수정 원칙

- 패치식 덧붙임 수정 금지
- 같은 기능을 여러 곳에 분산 금지
- 중복 코드 생성 금지
- 잘못 수정된 코드는 남기지 않고 원래 잘못된 위치에서 직접 정리
- 지도, 따라가기, 방향, 임시 상태, 종료 처리는 각각 하나의 주인 로직에서 관리

## 지도 기준

- 본앱의 다른 지도 카테고리와 동일하게 **Kakao Map**을 사용합니다.
- OpenStreetMap/Leaflet 지도는 사용하지 않습니다.
- 성지·성당·피정 지도, 일반 길찾기, 카카오내비 경로검색 기능은 넣지 않았습니다.
- 순례길 GPX 경로선, 스탬프 지점, 내 위치, 따라가기만 유지합니다.

## 포함된 순례길

- 한티가는길
- 서울순례길 1코스 · 말씀의 길: 실제 GPX
- 서울순례길 2코스 · 생명의 길: 실제 GPX
- 서울순례길 3코스 · 일치의 길: 실제 GPX
- 서울순례길 · 김대건 신부 치명 순교길: 실제 GPX
  - 서울순례길에 포함하되, 4코스로 번호를 붙이지 않음
- 도원동 테스트 GPX: 사용자 테스트용 GPX
- 회사 근처 테스트 GPX: 사용자 테스트용 GPX

## 서울순례길 지점 번호 기준

- 지도 숫자 마커는 `서울순례길.xlsx`의 `no/order` 번호와 좌표 기준입니다.
- 경로선은 업로드된 GPX 실제 경로 기준입니다.
- 즉, 숫자 지점은 엑셀 좌표, 걷는 파란 경로선은 GPX 좌표를 사용합니다.

## 이번 반영 사항

아래 피드백 20개를 한 번에 반영했습니다.

1. 독립 순례길 내비게이션 PWA 유지
2. 지도 엔진은 Kakao Map 기준 유지
3. OSM/Leaflet 제거 상태 유지
4. 성지·성당·피정 지도/일반 길찾기/카카오내비 기능 미포함 유지
5. 서울순례길 숫자 마커는 엑셀 번호·좌표 기준 유지
6. GPX 경로선 기준 따라가기 유지
7. 경로선 위 진행 방향 화살표 추가
8. 줌 아웃 시 화살표 1~3개 중심, 줌 인 시 적정 개수 표시
9. 정방향/역방향 전환 버튼 추가
10. 역방향은 GPX 순서, 스탬프 순서, 다음 지점, 진행률, 지난 길/앞길, 화살표를 모두 반대로 계산
11. 지나온 길과 앞으로 갈 길을 분리 표시
12. 실제 걸은 임시 이동 경로를 별도 선으로 표시
13. 지도 조작 중 자동 중심 복귀 지연을 15초로 변경
14. 상태 문구를 `지도 확인 중 · 15초 후 내 위치로 복귀` 기준으로 변경
15. 내위치 버튼은 즉시 현재 위치로 복귀
16. 종료 버튼 추가 및 `지금까지 이동한 경로가 삭제됩니다.` 확인창 적용
17. 확인 시 이번 따라가기 임시 이동 경로와 임시 상태 삭제
18. 짧은/긴 백그라운드 복귀 모두 8시간 미만이면 따라가기 복원
19. 8시간 이상 미사용 시 임시 따라가기 상태 자동 초기화
20. 완료 기록 저장 기능은 만들지 않고, 복귀용 임시 내비게이션 상태만 저장
21. 확대 상태에서 현재 화면에 경로선이 보이면 진행 방향 화살표가 최소 1개 이상 보이도록 화살표 배치 기준 보강
22. 정보카드는 기본 compact 상태로 낮게 시작하고, 손잡이를 누르거나 위/아래로 밀어서 쉽게 올리고 내릴 수 있게 정리
23. 정보카드 내부에서 `경로까지`와 `방향` 항목을 제거
24. 정보카드 지표를 운동시간, 운동거리, 진행률, 다음 지점까지 거리 4개로 단순화
25. 따라가기/정지/재시작/종료 버튼을 지도 위에 흩어두지 않고 하단 정보카드 안으로 정리
26. 따라가기 중에는 `정지`, 정지 상태에서는 `종료`와 `재시작` 버튼이 보이도록 조정

## 백그라운드 기준

PWA에서는 브라우저/OS가 허용하는 범위에서 `watchPosition`을 유지하고, 화면 복귀 시 현재 위치를 다시 확인해 경로 따라가기를 이어갑니다. 8시간 이상 위치 업데이트나 사용자 동작이 없으면 임시 따라가기 상태를 삭제합니다.

나중에 Android 본앱에 병합할 때는 등산 앱처럼 화면 꺼짐 상태에서도 안정적으로 추적하기 위해 Android Foreground Service와 백그라운드 위치 권한 구조를 붙이는 것이 맞습니다.

## v17 지도 초기화 수정

- 정보카드 단순화 작업 후 제거된 `updateDirectionMetric()` 호출이 `updateRouteHeader()`에 남아 있어 코스 선택 직후 JS가 중단되던 문제를, 해당 호출 위치에서 직접 삭제했습니다.
- 이 오류 때문에 Kakao Map 로딩까지 진행되지 않아 지도가 보이지 않았습니다.
- 지도/따라가기/정보카드 기능은 기존 주인 로직을 유지하고, 중복 함수나 패치성 우회 코드는 추가하지 않았습니다.


## v18 하단 정보카드 레이아웃 수정

- 사용자가 제시한 등산 앱 화면처럼 compact 상태의 하단 정보카드는 상태 문구 없이 4개 핵심 지표만 먼저 보이도록 정리했습니다.
  - 운동시간
  - 운동거리
  - 진행률
  - 다음 지점까지
- 정보카드 손잡이는 텍스트/막대형 대신 위아래 조작이 직관적인 보라색 꺾쇠 형태로 바꿨습니다.
- 역방향 버튼을 지도 우측 플로팅 버튼에서 제거하고, 시작/정지/재시작 버튼 옆의 하단 조작 영역으로 이동했습니다.
- 지도 위 플로팅 버튼은 내위치만 남겼습니다.
- 중복으로 남아 있던 빈 폴리라인 처리 호출을 정리했습니다.


## v19 back-arrow-sheet
- 진행 방향 표시를 삼각형이 아닌 일반 화살표(↑)로 변경했습니다.
- 하단 정보카드 compact 높이를 줄여 지도 표시 영역을 넓혔습니다.
- 정보카드 hidden 상태를 추가해 뒤로가기 1회 시 카드가 내려가고 종료 안내 배너가 표시되게 했습니다.
- 지도 화면에서 뒤로가기를 한 번 더 누르면 이번 따라가기 임시 이동 경로가 삭제되고 목록으로 돌아갑니다.
- 역방향 버튼은 시작 전 최초 준비 상태에서만 보이고, 시작/정지/재시작 상태에서는 숨깁니다.
- 기존 따라가기/정보카드/뒤로가기 주인 로직에서 직접 수정했습니다.

## v20 path-arrowhead
- 진행 방향 표시를 독립된 글자/삼각형 아이콘이 아니라 경로선 위에 얹히는 작은 화살촉 SVG로 정리했습니다.
- 기존 `renderDirectionArrowAtDistance()`와 `.direction-arrow` 주인 위치에서 직접 수정했고, 별도 중복 화살표 로직을 추가하지 않았습니다.
- 캐시 버전: `pilgrimage-route-nav-v20-path-arrowhead-20260707`
