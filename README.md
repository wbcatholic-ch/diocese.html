# 가톨릭길동무 순례길 내비게이션 독립 PWA

이 패키지는 원본 가톨릭길동무 앱과 분리한 순례길 내비게이션 테스트용 PWA입니다.

## 포함된 기능

- 순례길 목록
- 한티가는길 데이터 포함
- 서울순례길 4개 코스 데이터 포함: 1코스 말씀의 길, 2코스 생명의 길, 3코스 일치의 길, 김대건 신부 치명 순교길
- Kakao 지도에 GPX/대표 경로선 표시
  - 서울순례길 1코스 말씀의 길: 실제 GPX 반영
  - 서울순례길 2코스 생명의 길: 실제 GPX 반영
  - 서울순례길 3코스/김대건 신부 치명 순교길: GPX 확보 전 대표선 유지
- 스탬프 지점 표시
- 내 위치 확인
- 따라가기 모드
- 현재 위치와 GPX 경로선의 거리 계산
- 경로 위 / 근처 / 이탈 상태 표시
- 다음 지점 거리 표시
- Android 인앱 브라우저에서 Chrome으로 열기 버튼
- PWA 설치 및 Service Worker 캐시

## 제외한 기능

- 원본 앱의 성지·성당·피정의집 지도
- 원본 앱의 일반 길찾기 탭
- 카카오내비 경로검색 UI
- 숨겨진 테스트 지도 파일
- company/dowon 테스트 루프 데이터
- 한티가는길 데이터의 테스트 모드 필드
- 원본 앱 커버, 기도문, 교구, 나의 신앙생활 기능

## GitHub Pages 테스트 전 확인

Kakao Developers에서 JavaScript 키의 웹 플랫폼 도메인에 GitHub Pages 주소를 등록해야 지도가 정상 표시됩니다.
예: `https://사용자명.github.io`

## 새 순례길 추가 방식

`routes/새순례길.js` 파일을 추가하고 아래 형식으로 등록하면 됩니다. 실제 GPX가 없으면 임시로 대표선으로 표시하고, 실제 보행 경로가 확보되면 `routeSegments.points`만 교체하세요.

```js
window.PILGRIMAGE_ROUTES = window.PILGRIMAGE_ROUTES || [];
window.PILGRIMAGE_ROUTES.push({
  id: 'new-route',
  name: '새 순례길',
  startName: '출발지',
  finishName: '도착지',
  stamps: [],
  routeSegments: []
});
```

그리고 `index.html` 하단에 해당 데이터 파일 script를 한 줄 추가하세요.

## 이번 수정 메모

- 앱 아이콘 배경을 진한 남색으로 변경했습니다.
- 서울순례길 1코스 말씀의 길과 2코스 생명의 길은 업로드된 GPX 실제 경로선으로 교체했습니다.
- 서울순례길 3코스 일치의 길과 김대건 신부 치명 순교길은 아직 GPX가 없어 엑셀 좌표 대표선을 유지했습니다.
- 기존 앱의 일반 지도/길찾기/카카오내비 기능은 포함하지 않았습니다.
