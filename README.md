# 가톨릭길동무 순례길 내비게이션 독립 PWA

이 패키지는 원본 가톨릭길동무 앱과 분리한 순례길 내비게이션 테스트용 PWA입니다.

## 포함된 기능

- 순례길 목록
- 한티가는길 데이터 포함
- 서울순례길 4개 코스 데이터 포함: 1코스 말씀의 길, 2코스 생명의 길, 3코스 일치의 길, 김대건 신부 치명 순교길
- OpenStreetMap/Leaflet 기반 지도에 GPX/대표 경로선 표시
  - 서울순례길 1코스 말씀의 길: 실제 GPX 반영
  - 서울순례길 2코스 생명의 길: 실제 GPX 반영
  - 서울순례길 3코스 일치의 길: 실제 GPX 반영
  - 서울순례길 4코스 김대건 신부 치명 순교길: 실제 GPX 반영
- 지도 라이브러리 연결 실패 시 경로 전용 보기로 자동 전환
- 스탬프 지점 표시
- 내 위치 확인
- 따라가기 모드
- 현재 위치와 GPX 경로선의 거리 계산
- 경로 위 / 근처 / 이탈 상태 표시
- 다음 지점 거리 표시
- Android 인앱 브라우저에서 Chrome으로 열기 버튼
- PWA 설치 및 Service Worker 캐시

## 제외한 기능

- 원본 앱의 성지·성당·피정의집 지도
- 원본 앱의 일반 길찾기 탭
- 카카오내비 경로검색 UI
- 숨겨진 테스트 지도 파일
- company/dowon 테스트 루프 데이터
- 한티가는길 데이터의 테스트 모드 필드
- 원본 앱 커버, 기도문, 교구, 나의 신앙생활 기능

## GitHub Pages 테스트 전 확인

이번 버전은 GitHub Pages 독립 테스트가 먼저 되도록 Kakao 지도 의존을 제거하고 OpenStreetMap/Leaflet 기반으로 변경했습니다.
따라서 Kakao Developers 웹 플랫폼 도메인 등록 없이도 Chrome에서 지도가 먼저 떠야 합니다.

테스트 주소는 반드시 `https://` 주소로 여세요.
예: `https://사용자명.github.io/저장소명/`

## 새 순례길 추가 방식

`routes/새순례길.js` 파일을 추가하고 아래 형식으로 등록하면 됩니다. 실제 GPX가 없으면 임시로 대표선으로 표시하고, 실제 보행 경로가 확보되면 `routeSegments.points`만 교체하세요.

```js
window.PILGRIMAGE_ROUTES = window.PILGRIMAGE_ROUTES || [];
window.PILGRIMAGE_ROUTES.push({
  id: 'new-route',
  name: '새 순례길',
  startName: '출발지',
  finishName: '도착지',
  stamps: [],
  routeSegments: []
});
```

그리고 `index.html` 하단에 해당 데이터 파일 script를 한 줄 추가하세요.

## 이번 수정 메모

- 지도 로딩이 십자가 화면에서 멈추는 문제를 막기 위해 Kakao 지도 의존을 제거했습니다.
- 기본 지도 엔진을 OpenStreetMap/Leaflet로 변경했습니다.
- 외부 지도 라이브러리 연결이 실패해도 빈 화면에 멈추지 않고 경로 전용 SVG 보기로 자동 전환되게 했습니다.
- 내 위치/따라가기 계산을 지도 로딩 성공 이후가 아니라 순례길 선택 직후 준비하도록 바꿨습니다.
- 서비스워커는 내부 PWA 파일만 처리하고 외부 지도 요청은 가로채지 않습니다.
- CSS/JS 캐시 버전은 `20260707-osmfix1`을 유지하고, 서울순례길 데이터 버전을 `20260707-gpx4`로 올렸습니다.
- 앱 아이콘 배경은 진한 남색을 유지했습니다.
- 서울순례길 1·2·3코스는 업로드된 실제 GPX 경로선을 유지했습니다.
- 서울순례길 4코스 김대건 신부 치명 순교길은 업로드된 실제 GPX 경로선으로 교체했습니다. 경로 포인트 1,107개, 거리 약 13.2km입니다.
- 기존 앱의 일반 지도/길찾기/카카오내비 기능은 포함하지 않았습니다.
